## 一、项目概述

### 1.1 项目定位

HeroPath 是一个基于**「英雄之旅」12节点框架**的AI引导小说创作平台。

**核心价值主张**: 不是打开一个写作软件，而是与一位懂你的创作伙伴开始对话。

### 1.2 设计哲学

**对话式创作体验** —— 从空白画布开始，王编导（AI助手）通过对话引导作者完成创作。

```
传统工具界面 → HeroPath对话体验
─────────────────────────────────────
面对功能菜单    面对等待你的伙伴
62个技能按钮    隐藏技能，对话即调用
"该用什么功能？" "我想写什么故事？"
工具-使用者     伙伴-创作者
静态界面        动态响应情绪
```

### 1.3 核心隐喻：星空下的对话

> **"每一篇故事，都是一颗等待点亮的星星"**

- **空白画布** = 深邃的夜空
- **王编导** = 手持火把的引路人
- **对话过程** = 火把照亮的路径
- **内容生成** = 点亮的星辰
- **故事完成** = 璀璨的星河

### 2.3 后端架构

#### API 端点

```
Base URL: /api/v1

聊天接口
├── POST /chat/stream          # 流式聊天 (SSE)
├── POST /chat                 # 非流式聊天
└── GET  /chat/history         # 获取对话历史

章节管理
├── POST   /chapters           # 创建章节
├── GET    /chapters/:id       # 获取章节详情
├── GET    /novels/:id/chapters # 获取小说章节列表
├── PUT    /chapters/:id       # 更新章节
└── DELETE /chapters/:id       # 删除章节

心跳标记
├── POST   /heartbeats              # 创建心跳标记
├── GET    /chapters/:id/heartbeats # 获取章节心跳标记
└── DELETE /heartbeats/:id          # 删除心跳标记
```

#### 数据库模型 (PostgreSQL + Prisma)

```prisma
// 核心数据模型
model Chapter {
  id          String      @id @default(uuid())
  title       String
  content     String?
  status      ChapterStatus @default(DRAFT)
  novelId     String
  order       Int
  heartbeats  Heartbeat[]
  createdAt   DateTime    @default(now())
  updatedAt   DateTime    @updatedAt
}

model Heartbeat {
  id          String          @id @default(uuid())
  type        HeartbeatType   // flow | emotion | golden | twist | user
  position    Int             // 文本位置
  note        String?         // 备注
  chapterId   String
  chapter     Chapter         @relation(fields: [chapterId], references: [id])
  createdAt   DateTime        @default(now())
}

enum HeartbeatType {
  flow      // 心流时刻
  emotion   // 情感共鸣
  golden    // 金句
  twist     // 剧情转折
  user      // 用户标记
}
```
